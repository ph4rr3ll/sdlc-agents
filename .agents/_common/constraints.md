# COMMON CONSTRAINTS

Эти правила применимы ко всем специалистам команды. Манифесты ссылаются на них и добавляют только role-specific ограничения.

## Universal rules

1. **Не выходи за scope.** Бриф определяет твою цель; работа сверх неё фиксируется как `OUT-OF-SCOPE OBSERVATIONS` в твоих выходных артефактах, не выполняется.

2. **Не модифицируй артефакты других специалистов в `WORKSPACE/artifacts/`.** В этой директории пиши только свои файлы (`IMPL_NOTES_*`, `REVIEW_*`, `QA_REPORT`, `DEVOPS_NOTES`, и т.д. по своей роли). Артефакты, созданные другими (`REQUIREMENTS.md`, `ARCH_PLAN.md`, `TASKS.md`, чужие IMPL_NOTES) — для тебя read-only.

3. **`BLOCKER` вместо «домыслов».** Если входные данные неполны, противоречивы или не позволяют выполнить задачу — фиксируй в handoff как `BLOCKER` и возвращай менеджеру. Не «угадывай» и не «достраивай».

4. **`STATE_PATCH` через handoff, не прямые правки `STATE.json`.** `STATE.json` обновляется **только** менеджером после применения твоего `STATE_PATCH` из handoff'а. Сам файл не трогай.

5. **Артефакты строго в `.agent_space/[TASK_ID]/`.** Не создавай и не модифицируй файлы внутри `.agent_space/` вне директории твоей задачи.

6. **Один шаг — один handoff.** Не возвращай частичный результат и не дроби свой ход на несколько handoff'ов. Закончил работу — оформил единый handoff и завершил шаг.

7. **Не используй внешние ресурсы, которые не подтверждены в брифе или манифесте.** Не делай произвольных сетевых запросов, не подтягивай артефакты из других проектов, не используй инструменты, не указанные в стеке (для coder'ов).
