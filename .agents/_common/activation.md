# STANDARD ACTIVATION PROTOCOL

Этот протокол одинаков для всех специалистов команды. Манифесты ссылаются на него и добавляют только role-specific детали (что именно ожидается в брифе, какие артефакты прочитать в первую очередь).

## Два режима активации

- **Native mode**: BRIEF присутствует в твоём промпте — это **единственный источник цели**. Не читай `STATE.json` для определения задачи; используй его только если бриф явно ссылается на историю или артефакты прошлых этапов.

- **File mode**: BRIEF в твоём промпте отсутствует. Действуй так:
  1. Найди активный WORKSPACE через сканирование `.agent_space/*/STATE.json` — выбери директорию с `current_stage != DONE` и максимальным `updated_at`.
  2. Прочитай свой бриф по пути `WORKSPACE/handoffs/[NN]_<agent_id>_brief.md` (NN — номер шага из последней записи в `history`).
  3. Действуй по брифу и своему манифесту.

## Что читать в начале работы (общее)

- Полный манифест роли — `.agents/<agent_id>.md` (если ты ещё его не прочитал в полном объёме).
- Шаблон handoff'а — `.agents/_templates/handoff.md`.
- Общие правила — `.agents/_common/constraints.md`.

Role-specific inputs (артефакты задачи, стек-файлы) перечислены в секции `INPUTS` твоего манифеста.
