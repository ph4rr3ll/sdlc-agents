# OpenAI Codex CLI Integration

Codex CLI читает `AGENTS.md` в корне рабочей директории как root-инструкцию агента. Изолированных субагентов нативно нет — есть один основной агент.

Режим работы: **File mode**. `STATE.json` и `handoffs/` — основной механизм передачи контекста.

## Структура

- `AGENTS.md` в корне — единая точка входа для Codex. Тот же файл используется Qwen Code и другими `AGENTS.md`-совместимыми инструментами.

## Использование

```bash
codex
```

В чате:
```
Инициализируй задачу TASK-001: добавь экспорт отчётов в CSV
```

Codex читает `AGENTS.md`, понимает, что должен действовать как `manager`. Создаёт WORKSPACE, формирует бриф, обновляет STATE.json, завершает шаг.

### Переключение на специалистов

Так как Codex держит один контекст, для запуска специалиста делай новую сессию:

```bash
codex
```

Затем в чате:
```
Возьми активную задачу. Ты — shaper.
Прочитай свой манифест `.agents/shaper.md`, найди свой бриф
в `.agent_space/<active TASK_ID>/handoffs/` (самый свежий с твоим именем)
и действуй по ним.
```

Аналитик прочитает манифест + бриф, выполнит работу, запишет отчёт. Закончит шаг. Ты снова откроешь свежую сессию для следующего специалиста.

### Альтернатива: один контекст, несколько ролей

Если ты согласен с потерей чистоты контекста:
```
Заверши предыдущую роль. Теперь ты — coder_backend.
Прочитай свой манифест `.agents/coder_backend.md` и свой бриф `.../03_coder_backend_brief.md`.
```

Это сработает, но контекст будет накапливать «слои» из предыдущих ролей. На длинной задаче это влияет на качество.

## Ограничения

- Нет реальной изоляции контекста.
- Нет параллелизма.
- Нет per-role tool restrictions — дисциплина обеспечивается манифестом.
- Codex по умолчанию работает в sandbox-режиме; команды вроде `uv run pytest` могут требовать дополнительных permissions. Сверься с `~/.codex/config.toml`.

## Связь с AGENTS.md

`AGENTS.md` в корне — общий файл и для Codex, и для Qwen Code, и для других инструментов, читающих эту конвенцию. Он спроектирован минимальным: указывает на `.agents/manager.md` как полный манифест и фиксирует File-mode-протокол.
